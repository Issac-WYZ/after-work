# AGENTS.md

本文件写给 Codex 和后续自动化开发代理。所有开发行为必须遵守本项目的阶段范围、工程纪律和跨平台要求。

## 当前项目

- 游戏名：《下班以后》
- 类型：2D 像素风现代生活模拟游戏
- 引擎：Godot 4.6.x stable
- 脚本语言：GDScript
- 长期平台：macOS、Windows
- 当前版本：0.1 最小可玩原型

## 开发纪律

1. 每次只实现一个小模块。
2. 不允许擅自增加未要求功能。
3. 新想法进入 `BACKLOG.md`，不进入当前版本。
4. 每个阶段完成后必须说明如何测试。
5. 每个稳定阶段建议 git commit。
6. 当前 0.1 只允许使用 placeholder 占位素材。
7. 不允许引入版权不清楚的素材。
8. 不允许在 0.1 引入不必要的第三方插件。
9. 不允许为了未来可能需要而提前做复杂架构。
10. 每次改动后应保证项目仍可打开或运行。

## 0.1 范围限制

0.1 只实现：

- Apartment、Office、ConvenienceStore 三个基础场景。
- 玩家基础移动、碰撞、摄像机跟随。

- GameState 全局状态。
- 状态 HUD。
- 工作小游戏。

- 技能系统
- 多职业

- 天气

## 跨平台规则

1. 不允许写死 Windows 或 macOS 专属路径。
2. 项目资源路径必须使用 Godot 的 `res://`。
3. 用户数据、存档、配置和缓存必须使用 Godot 的 `user://`。
4. 输入系统必须使用 Godot InputMap。
5. 不在玩法逻辑中直接绑定单一平台键盘逻辑。
6. UI 必须适配不同屏幕比例和窗口尺寸。
7. 音频、字体、图片资源必须使用 Godot 可跨平台加载的格式。
8. 每个重要版本都要测试 macOS 和 Windows 导出。
9. 不要引入只支持单一平台的插件。

## 资源规范

- 0.1 只使用 placeholder 占位素材。
- 图片优先使用 `.png`。
- 短音效优先使用 `.wav`。
- 音乐优先使用 `.ogg`。
- 字体优先使用 `.ttf` 或 `.otf`。
- 第三方素材必须记录来源、作者、许可证和链接。
- 版权不明确的素材不得进入仓库。

## 提交规范

建议使用以下提交类型：

- `docs:` 文档变更
- `chore:` 工程配置或目录整理
- `feat:` 新功能
- `fix:` 修复问题
- `test:` 测试和验收
- `refactor:` 不改变行为的结构调整

示例：

- `docs: add v0.1 product design requirements`
- `chore: initialize godot project settings`
- `feat: add player movement`
- `test: verify v0.1 daily loop`
