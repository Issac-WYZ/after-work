# 0.1 Test Checklist

本清单用于 0.1 每个阶段的验收。每次完成稳定阶段后，应记录测试结果并建议 git commit。

## 工程基础

- [ ] 项目可被 Godot 4.6.x stable 打开。
- [ ] 项目没有写死 macOS 专属路径。
- [ ] 项目没有写死 Windows 专属路径。
- [ ] 项目资源路径使用 `res://`。
- [ ] 用户数据、存档、配置和缓存使用 `user://`。
- [ ] 输入动作通过 Godot InputMap 定义。
- [ ] 没有引入只支持单一平台的插件。
- [ ] 0.1 只使用 placeholder 占位素材。
- [ ] 第三方素材来源和许可证已记录。

## 显示与 UI

- [ ] 默认窗口比例为 16:9。
- [ ] UI 在 16:9 下不重叠。
- [ ] UI 在 16:10 下不重叠。
- [ ] UI 在 4:3 下不重叠。
- [ ] 窗口缩放后核心 UI 仍可读。
- [ ] 像素素材没有明显模糊或拉伸。

## 0.1 核心循环

- [ ] 玩家可以在 Apartment 移动。
- [ ] 玩家移动有基础碰撞。
- [ ] 摄像机跟随玩家。
- [ ] HUD 显示 Day、Time、Money、Energy、Mood、Stress。
- [ ] Apartment 可以切换到 Office。
- [ ] Apartment 可以切换到 ConvenienceStore。
- [ ] Office 可以返回 Apartment。
- [ ] ConvenienceStore 可以返回 Apartment。
- [ ] 玩家可以进入工作小游戏。
- [ ] 完成 5 次正确输入后获得工资 80。
- [ ] 工作完成后 Energy -30。
- [ ] 工作完成后 Stress +20。
- [ ] 工作完成后 Time 变为 18:00。
- [ ] 玩家可以购买便当。
- [ ] 玩家可以购买咖啡。
- [ ] 玩家可以购买甜品。
- [ ] 商品购买后 Money 和状态变化正确。
- [ ] 玩家可以在 Apartment 睡觉。
- [ ] 睡觉后 Day +1。
- [ ] 睡觉后 Time 回到 07:00。
- [ ] 睡觉后 Energy 恢复到 100。
- [ ] 睡觉后 Stress 正确降低且不低于 0。
- [ ] 状态变化后 UI 立即刷新。

## 导出测试

- [ ] macOS 导出成功。
- [ ] macOS 导出版可启动。
- [ ] Windows 导出成功。
- [ ] Windows 导出版可启动。
- [ ] 导出测试结果已记录到 `DEVLOG.md`。
